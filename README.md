# Business Sales Performance Analytics

## 📌 Project Overview

**Business Sales Performance Analytics** is a data analytics project developed to analyze and visualize business sales data using interactive dashboards. The project focuses on transforming raw transactional data into meaningful insights that help organizations monitor performance, identify trends, and make data-driven business decisions.

The dashboard provides a comprehensive view of sales performance by analyzing key business metrics such as revenue, profit, customer behavior, product performance, and regional sales distribution. Through effective data storytelling and visualization techniques, stakeholders can easily understand business performance and identify opportunities for growth.

---

## 🎯 Objectives

* Analyze overall business sales performance.
* Monitor Key Performance Indicators (KPIs).
* Identify sales trends and seasonal patterns.
* Evaluate product and regional performance.
* Understand customer purchasing behavior.
* Generate actionable insights for strategic decision-making.

---

## 📂 Dataset Description

The project utilizes the **Online Retail Dataset**, which contains transactional records of an online retail company. The dataset consists of customer purchase information collected over a specific period.

### Dataset Features

| Feature     | Description                            |
| ----------- | -------------------------------------- |
| InvoiceNo   | Unique identifier for each transaction |
| StockCode   | Unique identifier for products         |
| Description | Product name or description            |
| Quantity    | Number of units purchased              |
| InvoiceDate | Date and time of transaction           |
| UnitPrice   | Price per product unit                 |
| CustomerID  | Unique customer identifier             |
| Country     | Country where the customer belongs     |

## 🧹 Data Cleaning and Preparation

The following preprocessing steps were performed:

* Removed duplicate transaction records.
* Handled missing and null values.
* Removed canceled orders and invalid transactions.
* Corrected inconsistent data entries.
* Converted date columns into proper datetime format.
* Created calculated columns for advanced analysis.
* Filtered negative quantities and prices.
* Standardized column names and formats.
